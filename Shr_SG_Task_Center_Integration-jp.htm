SAP Task Center Integration with Concur Solutions

1           アクセス許可

この機能へのアクセス権があるかどうかは場合によります。たとえば、特定のグループに対してのみアクセス権がある、または読取専用で作成や編集はできないなど、限定的なアクセス権を持っている場合があります。

管理者はこの機能を使用する必要がありますので、適切なアクセス許可がない場合は、社内の SAP Concur 管理者にお問い合わせください。

さらに、このガイドに記載されている作業は SAP Concur にのみ許可されているものもあります。必要に応じて SAP Concur クライアント サポートにご依頼ください。

2           概要

SAP Task Center は、Web を介してユーザーに提供される一元化されたソリューションを通じて、管理上の負担を軽減し、運用プロセスを改善するとともに、承認プロセス内の洞察を強化でき、SAP ポートフォリオ内の複数のソリューションを使用するお客様のタスクの遂行を支援します。

SAP Task Center と SAP Concur ソリューションの統合により、Task Center に経費精算レポートや申請の承認を取り込むことができます。

この統合では、以下の SAP Concur ソリューションがサポートされます。

·        Concur Expense（Professional/Premium Edition）

·        Concur Request（Professional/Premium Edition）

2.1     機能

統合により、以下を行うことができます。

·        承認待ちの経費精算レポートの表示

·        経費精算レポートの承認

·        経費精算レポートの差戻し

·        承認待ちの申請の表示

·        申請の承認

·        申請の差戻し

統合では、以下のユーザー ロールを有する承認者がサポートされます。

·        経費承認者

·        申請の承認者

·        承認権限者

·        原価対象の承認者

·        予算承認者

2.1.1         対象外の項目

SAP Task Center と Concur ソリューションの統合では、Concur Expense および Concur Request の以下の機能はサポートされません。

·        代理による承認

·        処理者による承認

·        仮払の承認

·        部分承認（承認金額の修正）

·        SAP Task Center からレポート/申請の SAP Concur Mobile アプリ表示へのディープ リンク

·        レポート/申請に関連する予算の表示

·        走行経費の Google Maps 情報の表示

·        承認済ワークフロー ステータスのレポート/申請の表示

·        レポート/申請の次の承認者の選択（承認して転送）

さらに、Task Center での Concur Expense/Concur Request タスクで、Task Center と同じ言語がサポートされない可能性もあります。サポートされない言語をお使いの場合は、既定の米国英語 (en-us) でタスクが表示されます。

2.2     利用可能な SAP Task Center 文書

本ガイドでは、Concur Expense および Concur Request システムで SAP Task Center との統合を設定するために必要なタスクに焦点を合わせます。SAP Task Center の設定については、SAP Task Center 文書をご参照ください。

3           設定

3.1     概要

SAP Task Center では、Concur Expense および/または Concur Request と SAP Task Center の両方で設定ステップが必要です。本ガイドでは、Concur Expense および/または Concur Invoice での統合を設定および有効化するためのステップに関する情報を記載します。

&   SAP Task Center での設定については、本ガイドの「2.2 利用可能な SAP Task Center 文書」セクションをご参照ください。

3.2     ユーザー複製シナリオ

SAP Task Center と Concur ソリューションの統合では、お客様の既存の環境やアイデンティティ プロバイダに基づき、ユーザー複製の複数の設定をサポートしています。ユーザー情報の中心となるのは一意の userUUID です。これは、SAP Concur ソリューションを含め、Task Center を提供するすべての SAP アプリケーションで一貫している必要があります。

お客様は、SAP Cloud Identity Services を独自のアイデンティティ プロバイダとして使用するのか、サードパーティのアイデンティティ プロバイダを使用するのかを選択する必要があります。SAP Task Center と Concur ソリューションの統合では、以下に示すように、どちらの設定もサポートが可能です。すべての複製モデルで、SAP Task Center と Concur ソリューションの統合では、各ユーザーの userUUID について SAP Identity Authentication Service (IAS) を参照します。

NOTE: SAP Cloud Identity Services が、お客様の SAP 環境でベスト プラクティスのユーザー複製方法です。

NOTE: IAS/IPS をまだご使用でない場合は、IAS/IPS へ移行する前に、Production Sandbox Environment (PSE) テスト エンティティを使用してユーザー複製をテストすることをお勧めします。

3.2.1         SAP Cloud Identity Services の使用

SAP Cloud Identity Services を実装予定またはすでに実装済みのお客様は、Identity Provisioning Service (IPS) と Identity Authentication Service (IAS) を使用して、各ユーザーのユーザー データ（userUUID を含む）の作成、維持、伝播を行います。このユーザー データを受け入れるには、SAP Concur エンティティにおいて、SAP IAS が SSO に設定され、SAP IPS の IdP ターゲットとして SAP Concur エンティティが設定されている必要があります。

&   詳細については、本ガイドの「2.3.3 SAP Concur シングル サインオン ガイドおよびアイデンティティ プロバイダ ターゲット ガイド」セクションをご参照ください。

3.2.2         サードパーティのアイデンティティ プロバイダの使用

SAP IAS をプロキシとして、SAP Task Center でサードパーティのアイデンティティ プロバイダ (IdP) を引き続き使用できます。この場合、SAP IAS へのユーザーの複製はお客様が行います。お客様自身で userUUID を作成するか、または SAP IAS で作成する必要があります。

&   詳細については、https://help.sap.com/docs/SAP_CLOUD_IDENTITY/b95c3d5bab324a3a8409eee5267a5b75/a04611df60404a248a7a8089c85b9761.html をご参照ください。

このステップ後、以下の 2 つの方法で、SAP Concur ソリューションにユーザー情報を複製できます。お客様の環境内のすべての SAP アプリケーションで userUUID が一致していることを確認してください。

直接的な方法: 以下のいずれかのユーザー インポート方法により、userUUID（およびその他のユーザー データ）を SAP Concur ソリューションに直接送信します。

·        アイデンティティ v4 API

·        UPS v4 API

·        フラット ファイル インポート（従業員インポート レコード 315）

&   詳細については、本ガイドの「2.3.4 SAP Concur ユーザー インポートおよびプロビジョニング文書」セクションに記載の文書をご参照ください。

間接的な方法: userUUID を送信するか、または IAS で生成することにより、IAS で新しいユーザーを作成できます。これにより、IPS がソース システムとして IAS からユーザー データ（userUUID を含む）を取り込み、ターゲット システムとして SAP Concur ソリューションにユーザー データを送信します。さらに、お客様が以下のいずれかのユーザー インポート方法を使用して、より詳しいユーザー データを SAP Concur ソリューションに直接提供できます。

·        アイデンティティ v4 API

·        UPS v4 API

·        フラット ファイル インポート

&   詳細については、本ガイドの「2.3.4 SAP Concur ユーザー インポートおよびプロビジョニング文書」セクションに記載の文書をご参照ください。

3.3     前提条件

3.3.1         SAP Concur

·        お客様が Data Protection Agreement (DPA) に署名している必要があります。

·        SAP Concur の新規または既存のお客様で、Concur Expense Professional/Premium Edition および/または Concur Request Professional/Premium Edition をお使いである必要があります。

·        SAP の新規または既存のお客様で、以下の SAP システムをお使いである必要があります。

¨       SAP IAS および SAP IPS テナントを持つ SAP Cloud Identity Service

·        統合の設定を行う SAP Concur 管理者は以下のアクセス許可が付与されている必要があります。

¨       「Expense の構成管理者」または「経費精算管理者」のユーザー ロール（経費ワークフローを表示および修正する権限を付与するため）

¨       「Request の構成管理者」（申請ワークフローを表示および修正する権限を付与するため）

¨       「Web サービス管理者」ユーザー ロール（会社リクエスト トークンおよび会社 UUID を生成する権限を付与するため）

·        SAP Concur エンティティにおいて、SAP IAS が SSO に設定され、SAP IPS の IdP ターゲットとして SAP Concur エンティティが設定されている必要があります。

·        SAP Concur エンティティで米国英語が有効になっている必要があります。SAP Task Center では既定で米国英語がフォールバック言語として設定されています。

·        本ガイドの「ユーザー複製シナリオ」セクションで詳述するように、ユーザー複製方法を選択および設定している必要があります。

·        identity.user.sap.read、identity.user.sap.write のアイデンティティ v4 API スコープが必要です。

3.4     ワークフローの設定

SAP Concur 実装コンサルタントは、お客様の Concur Expense および/または Concur Request の設定に応じて、以下のステップの（すべてではないにしても）多くを行います。

Task Center は Concur Expense および/または Concur Request に対して個別で有効になります。

3.4.1         Concur Expense: 経費ワークフローの設定の更新

SAP Concur 管理者は、[管理] > [経費精算] > [ワークフロー] で SAP Task Center との統合で経費精算レポート承認を有効にします。

[設定] タブで [SAP Concur 承認を SAP Task Center と接続] チェック ボックスを選択します。

&   詳細情報は、設定ガイド「Concur Expense: ワークフロー（概要）」をご参照ください。

この設定を選択すると、SAP Task Center との統合ですべての経費精算レポートが利用可能になります。

3.4.2         Concur Request: 申請ワークフローの設定の更新

SAP Concur 管理者は、[管理] > [事前申請] > [ワークフロー] で SAP Task Center との統合で申請の承認を有効にします。

[設定] タブで [SAP Concur 承認を SAP Task Center と接続] チェック ボックスを選択します。

&   詳細情報は、設定ガイド「Concur Request: ワークフロー - 概要」をご参照ください。

この設定を選択すると、SAP Task Center との統合ですべての申請が利用可能になります。

3.5     SAP Concur 会社リクエスト トークン

SAP Concur 管理者は会社リクエスト トークンを生成して、SAP Task Center の複製先として Concur Expense および/または Concur Request を設定する際に使用することができます。

4   SAP Concur 会社リクエスト トークンを入手するには:

1.    「会社リクエスト トークン セルフサービス ツール」ページで詳述するステップに従い、会社リクエスト トークンを生成します。ご利用のデータ センターに適切な Task Center のアプリ ID を使用してください。

データ センター

アプリ ID

US1、EU1

a8a7ec4d-828a-4c98-9fea-9cc273c0558d

US2、EU2

d43da2d2-c0ac-4477-8de9-6c4b11057705

NOTE: 使用しているデータ センターが不明な場合は、SAP Concur 実装コンサルタントにお問い合わせください。

2.    [会社リクエスト トークンが作成されました] ダイアログから、会社 UUID と会社リクエスト トークンの両方をコピーして保存します。

NOTE: 会社リクエスト トークンは 24 時間経過すると失効します。Task Center の複製先の設定で 24 時間以内に会社リクエスト トークンを使用しなかった場合、このステップを繰り返し、新しいトークンを申請する必要があります。

3.    SAP Task Center 文書に記載のステップに従い、会社 UUID と会社リクエスト トークンを使用して、SAP Concur の複製先を作成します。
